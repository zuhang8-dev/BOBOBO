| 異常現象 | 位置 |

## 根本原因分析

### 🔴 主因：`decision_dataset.csv` 不存在

ROI 模擬器的所有 API 端點都依賴此檔案：

```
app.py → _load_decision_df()
       → data/processed/decision_dataset.csv  ← 缺少！
```

API 收到請求後直接拋出 HTTP 404，前端收到錯誤後什麼都不渲染，KPI 停在初始值 `--`。

**此檔案需由 `scripts/build_decision_dataset.py` 產生**，而該腳本需要原始資料集 `DataCoSupplyChainDataset.csv` 放在 `data/raw/`，但 `data/raw/` 目錄只有 `.gitkeep`（原始資料不入版控）。

**資料流圖**：
```
DataCoSupplyChainDataset.csv   ← 組員未放入 data/raw/
         ↓  build_decision_dataset.py
decision_dataset.csv           ← 因此不存在
         ↓  /api/roi/summary, /api/roi/portfolio …
ROI 模擬器 UI                  ← 全部顯示 -- / NaN
```

### 🟡 次因 1：`_fmtMoney` / `_fmtPct` 缺少 null/NaN 防護

```js
// 修復前（roi_simulator.js）
const _fmtMoney = (v) => (v < 0 ? '-$' : '$') + Math.abs(Math.round(v)).toLocaleString();
const _fmtPct = (v) => (v * 100).toFixed(1) + '%';
```

當 API 回傳 `undefined` 或 `null` 時：
- `Math.round(undefined)` → `NaN`
- `(undefined * 100).toFixed(1)` → `NaN`
- 畫面顯示「`$非數值`」和「`NaN%`」

### 🟡 次因 2：KPI 卡片 CSS 結構錯誤

`.kpi-card` 的全域樣式是為「左側 icon + 右側文字」設計的橫向 flex：

```css
.kpi-card {
  display: flex;
  align-items: center;  /* 橫向排列 */
  gap: 20px;
}
```

但 ROI 模擬器的 KPI 卡片沒有包 `.kpi-icon-wrap`＋`.kpi-info`，直接把 `.kpi-label`、`.kpi-value`、`.kpi-foot` 三個 div 放進去，導致橫向排列、文字擠壓、金額字串溢出。

## 修復過程（按時序）

### Step 1：前端 JS 修復（NaN 防護）

**修改檔案**：`static/roi_simulator.js`

```js
// 修復後
const _fmtMoney = (v) => {
  const n = parseFloat(v);
  if (v == null || isNaN(n)) return '—';
  return (n < 0 ? '-$' : '$') + Math.abs(Math.round(n)).toLocaleString();
};
const _fmtPct = (v) => {
  const n = parseFloat(v);
  if (v == null || isNaN(n)) return '—';
  return (n * 100).toFixed(1) + '%';
};
```

**效果**：API 未就緒時顯示 `—` 而非崩潰的「`$非數值`」。

## 問題歸因（組員改動）

| 問題 | 歸因 |
|------|------|
| `data/raw/DataCoSupplyChainDataset.csv` 未放入版控環境 | 組員在自己電腦跑 pipeline 後，未留下使用說明也未確認新環境是否有原始資料 |
| `profit_lightgbm_model.txt` 損壞無法載入 | 模型檔可能在不同機器間傳輸時損壞，或用不相容版本的 lightgbm 存檔 |
| `profit_train_ready.csv` 只有 55 筆 | 組員以小 mock 資料測試後未更新，留下不正確的訓練資料殘留 |
| `score_profit()` 未考慮新模型無 categorical 設定 | 寫死了舊模型的 categorical 轉換，缺乏向前相容設計 |

## 後續建議

1. **建立 `setup_and_run.bat` 明確說明**：需先將 `DataCoSupplyChainDataset.csv` 放入 `data/raw/` 才能執行 ROI 模擬器相關功能。
2. **在 `build_decision_dataset.py` 加入友善提示**：當 `decision_dataset.csv` 不存在時，API 應回傳更明確的錯誤訊息（目前已回 404 + 說明文字，前端可考慮顯示 setup 引導）。
3. **模型版本管控**：記錄訓練時所用的 lightgbm 版本，在 `profit_model_metrics.json` 中加入 `lightgbm_version` 欄位。
4. **收益模型重訓後應重跑**：每次 `profit_model_pipeline.py` 重訓後，需重新執行 `scripts/build_decision_dataset.py` 以更新 decision_dataset。
