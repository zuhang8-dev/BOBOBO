## 二、地圖 GeoJSON 端點修復過程

### 原始程式碼（有問題）
```python
@app.get("/api/geojson/countries")
async def get_countries_geojson():
    import httpx
    url = "https://raw.githubusercontent.com/datasets/geo-countries/master/data/countries.geojson"
    async with httpx.AsyncClient(timeout=30) as client:
        res = await client.get(url)
        return JSONResponse(content=res.json())
```

### 修復演進

#### 選項 2：改用 `urllib`（保留原始代理設計）
```python
from urllib.request import urlopen
with urlopen(url, timeout=30) as response:
    data = json.loads(response.read().decode("utf-8"))
    return JSONResponse(content=data)
```
- 移除 `httpx` 依賴，改用 Python 標準函式庫
- 完全保留「後端代理解決 CORS」的設計意圖

#### 選項 3：加入本地 fallback
```python
except Exception:
    local_path = BASE_DIR / "data" / "countries.geojson"
    if local_path.exists():
        with open(local_path, "r", encoding="utf-8") as f:
            return JSONResponse(content=json.load(f))
```
- 網路或遠端失敗時，自動讀本機 `data/countries.geojson`

#### 方案 A：記憶體快取 + HTTP Cache-Control（最終採用，符合國際規格）

新增三層優化：

1. **啟動預載**：伺服器啟動時將本地 GeoJSON 預載進記憶體
2. **伺服器記憶體快取**：`_GEOJSON_CACHE` 全域變數，第二次起直接回傳
3. **HTTP Cache-Control 標頭**：`Cache-Control: public, max-age=86400`，瀏覽器快取 24 小時

```python
_GEOJSON_CACHE = None

@app.on_event("startup")
async def startup_event():
    global _GEOJSON_CACHE
    _geojson_local = BASE_DIR / "data" / "countries.geojson"
    if _geojson_local.exists():
        with open(_geojson_local, "r", encoding="utf-8") as f:
            _GEOJSON_CACHE = json.load(f)
        print("[GeoJSON] 已預載至記憶體。")
```

| 情境 | 改動前 | 改動後 |
|------|--------|--------|
| 伺服器啟動 | 無預載 | 自動預載進記憶體 |
| 第一次打開地圖 | 遠端抓取（慢） | 記憶體回傳（快）|
| 之後每次打開 | 每次重抓 | 瀏覽器快取（零延遲）|
| 網路斷線 | 地圖壞掉 | fallback 本地檔案 |

## 四、LLM 設定頁面 Bug 分析

完整審查 `index.html` + `app.py` 的 LLM 設定功能，發現以下問題：

### Bug 1 🔴（已修）：`updateLLMCustomModelState` 搶走焦點
**位置**：`index.html` `updateLLMCustomModelState()`  
**原因**：`loadLLMSettings()` 程式化觸發時，`focus()` 無條件執行，焦點跳到自訂輸入框  
**修法**：
```js
// 改前
if (isCustom) customInput.focus();
// 改後
if (isCustom && document.activeElement === modelInput) customInput.focus();
```

### Bug 2 🟡（已修）：清除 API Key 無警告
**位置**：`index.html` `saveLLMSettings()`  
**原因**：取消勾選「保留 Key」後直接儲存，靜默清除 API Key  
**修法**：加入 `confirm()` 確認對話框

### Bug 3 🟡（待修）：DPAPI 解密失敗靜默丟失 Key
**位置**：`app.py` `read_llm_runtime_config()`  
**原因**：`except Exception: return {}` 吞掉解密錯誤，前端顯示「尚未設定」但無錯誤說明  
**修法**：分離解密 try/except，回傳 `api_key_decrypt_error` 欄位，前端顯示 ⚠️ 警告

### Bug 4 🟡（待修）：切換 Provider 後自訂 URL 遺失
**位置**：`index.html` `handleLLMProviderChange()`  
**原因**：切換 provider 時直接覆寫為預設 URL，使用者自訂值消失  
**修法**：加入 `_llmProviderUrlCache` 物件，按 provider 分別儲存 URL

### Bug 5 🟠（待修）：Claude 模型 ID 可能無效
**位置**：`index.html` `LLM_MODEL_OPTIONS.claude`  
**原因**：`claude-opus-4-1-20250805`、`claude-sonnet-4-20250514` 等 ID 與 Anthropic 現行命名不符  
**修法**：更新為 `claude-opus-4-8`、`claude-sonnet-4-6`、`claude-haiku-4-5-20251001` 等正確 ID
