## 1. 地圖無法顯示問題診斷

**使用者提問：** 用 Git Bash 執行 `D:\anaconda_envs\edis_logistics_optimization`，但無法顯示全球物流延遲風險熱力圖。

**初步診斷的三個可能原因：**

### 問題 1：GeoJSON 從 GitHub 即時抓取
`app.py` 第 1535 行的後端 endpoint 每次都要向 GitHub 抓取世界地圖資料：
```python
url = "https://raw.githubusercontent.com/datasets/geo-countries/master/data/countries.geojson"
```
網路不穩定或 GitHub 被封鎖時，地圖完全無法顯示。前端 `region_map.js` 只顯示「地圖資料載入失敗，請檢查網路連線」。

### 問題 2：Git Bash 無法正確啟動 Conda 環境
Git Bash 預設不支援 `conda activate`：
```bash
conda activate Fastapp  # ❌ 失敗
source activate Fastapp  # ✓ 正確
conda run -n Fastapp uvicorn app:app --port 8000  # ✓ 正確
```

### 問題 3：Leaflet.js 從 CDN 載入
`index.html` 從 `unpkg.com` 載入地圖框架，CDN 被封鎖時整個地圖無法初始化。

**提供的三個解決方法：**
- 方法 A：把 GeoJSON 存到本地（最根本）
- 方法 B：在 Git Bash 中正確啟動 conda 環境
- 方法 C：直接用 `setup_and_run.bat`

## 3. 根本原因確認與修復

**使用者回報：** 嘗試方案 C（setup_and_run.bat）後，截圖顯示「地圖資料載入失敗，請檢查網路連線」，Leaflet 底圖（淡藍色背景）正常但無國家邊界。

**測試過程：**
```
curl http://localhost:8000/api/geojson/countries → 500 Internal Server Error
curl https://raw.githubusercontent.com/.../countries.geojson → 200 OK（2.7 秒）
```

**確認根本原因：`httpx` 未安裝在 Fastapp 環境中**

```
ModuleNotFoundError: No module named 'httpx'
```

問題根源在 `app.py` 的 endpoint 設計：
```python
@app.get("/api/geojson/countries")
async def get_countries_geojson():
    import httpx          # ← 直接拋出 ImportError（在 try 區塊外）
    try:
        ...               # ← 永遠執行不到
    except Exception as e:
        ...               # ← 攔截不到 import 的錯誤
```

`environment.yml` 雖然有列 `httpx>=0.27`，但 Fastapp 環境實際上未安裝（環境建立過程可能中斷了 pip 區段）。

**修復方式：**

1. **下載 GeoJSON 到本地**（14MB）：
```powershell
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/datasets/geo-countries/master/data/countries.geojson" `
    -OutFile "D:\anaconda_envs\edis_logistics_optimization\data\countries.geojson"
```

2. **修改 `app.py`**，改為讀本地檔案（移除 httpx 依賴）：
```python
@app.get("/api/geojson/countries")
async def get_countries_geojson():
    geo_path = BASE_DIR / "data" / "countries.geojson"
    if not geo_path.exists():
        raise HTTPException(status_code=404, detail="countries.geojson 不存在，請重新下載")
    try:
        with open(geo_path, "r", encoding="utf-8") as f:
            return JSONResponse(content=json.load(f))
    except Exception as e:
        raise HTTPException(status_code=500, detail=f"GeoJSON 讀取失敗：{e}")
```

## 4. Port 衝突排除

**使用者回報：** 重啟伺服器時出現：
```
[Errno 10048] error while attempting to bind on address ('127.0.0.1', 8000)
[ERROR] Server failed to start. Port 8000 may already be in use.
```

**原因：** 舊的 uvicorn 程序（PID 11124）仍在執行，佔用 Port 8000。伺服器啟動時沒有使用 `--reload` 旗標，無法自動偵測檔案變更。

**解決：**
```powershell
Stop-Process -Id 11124 -Force
```
Port 釋放後（TIME_WAIT 狀態為正常 TCP 斷線殘留），重新執行 `setup_and_run.bat` 即可。

## 5. 問題完整流程解說

### 問題鏈

```
httpx 未安裝於 Fastapp conda 環境
  → app.py endpoint 執行到 import httpx 拋出 ImportError
    → ImportError 在 try 區塊外，未被 except 攔截
      → FastAPI 回傳 500 Internal Server Error
        → 前端 fetch('/api/geojson/countries') 拿到錯誤
          → 顯示「地圖資料載入失敗，請檢查網路連線」
```

### 修復摘要

| 步驟 | 做了什麼 | 為什麼這樣做 |
|------|----------|-------------|
| 下載 GeoJSON | `data/countries.geojson`（14MB） | 不再依賴網路，執行更穩定 |
| 修改 `app.py` | 改用 `json.load()` 讀本地檔案 | 移除 `httpx` 依賴，用 Python 內建標準函式庫 |
| 終止舊程序 | `Stop-Process -Id 11124` | 釋放 Port 8000 讓新伺服器啟動 |
